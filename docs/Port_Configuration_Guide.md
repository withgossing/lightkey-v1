# 🌐 SSO 시스템 포트 설정 가이드 (Port Configuration Guide)

**문서 작성자:** Architect Agent / DevOps Agent
**버전:** 1.0.0

본 SSO 시스템은 도커(Docker) 컨테이너 기반으로 동작하며, 개발 환경 및 운영 환경에서 발생할 수 있는 포트 충돌(기본 80, 443, 3000, 5432 등과의 충돌)을 방지하기 위해 **10000번대** 이상의 사용자 정의(Custom) 연속 포트 대역을 사용합니다. 

권장하는 포트 대역은 **`18000 ~ 18010`** 입니다. (동위 시스템과 충돌 확률이 극히 낮음)

---

## 서비스별 추천 포트 매핑 (Recommended Ports)

| 서비스 명칭 (컨테이너 명) | 기본 포트 | 권장 커스텀 포트 | 포트 매핑 (Host:Container) | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **PostgreSQL 18** (`sso-db`) | 5432 | **18001** | `18001:5432` | 사원정보 및 OAuth 클라이언트 메인 DB |
| **Redis 8** (`sso-redis`) | 6379 | **18002** | `18002:6379` | 세션/Refresh Token, 인가 코드 캐시 DB |
| **NestJS 11 Backend** (`sso-api`) | 3000 | **18003** | `18003:18003` | SSO 인증 및 토큰 발급 (내부 포트도 동일하게 변경 커스터마이징 권장) |
| **Next.js 16 Frontend** (`sso-web`) | 3000 | **18004** | `18004:18004` | 통합 로그인 창 및 어드민 대시보드 UI (내부 포트 동일하게 변경 권장) |
| **Nginx / API Gateway** (`sso-proxy`) | 80/443 | **18000** / **18443** | `18000:80` / `18443:443` | (옵션) 서비스들을 단일 진입점으로 묶기 위한 프록시 서버 |

---

## 설정 적용 방안

1.  **Docker Compose (`docker-compose.yml`):**
    *   위 매핑 규칙에 따라 `ports:` 속성을 정의합니다. (예: `ports: ["18001:5432"]`)
2.  **환경 변수 (`.env`):**
    *   `DB_PORT=18001`
    *   `REDIS_PORT=18002`
    *   `API_PORT=18003`
    *   `WEB_PORT=18004`
    이러한 값들을 소스코드나 설정 파일(Config)에 하드코딩하지 않고 환경변수로 주입하여 실행 환경에 유연하게 대처합니다.
