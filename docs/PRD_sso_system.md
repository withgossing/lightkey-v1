# 📄 기능 요구사항 정의서 (PRD) - SSO 시스템 버전

**프로젝트명:** 사원정보 기반 싱글 사인온(SSO) 통합 인증 관리 애플리케이션 (Lightkey SSO System)
**문서 작성자:** PM Agent
**버전:** 1.0.0
**목적:** 한 번의 로그인(Single Sign-On)으로 기업 내 다양한 사내 시스템 및 그룹웨어에 안전하고 편리하게 접근할 수 있는 중앙 집중형 인증(Identity Provider, IdP) 체계를 구축한다.

---

## 1. 사용자 역할 (Roles) 및 대상
- **일반 사원 (ROLE_USER):** 식별 가능한 사번과 비밀번호(필요 시 2FA)로 중앙 인증 시스템에 로그인하여 가입된 통합 사내 서비스들을 추가 로그인 없이 이용할 수 있는 사용자.
- **최고 관리자 (ROLE_ADMIN):** 인증 시스템의 사용자 세션을 모니터링하고, 토큰 강제 만료, 사용자 권한 및 연동 애플리케이션(Service Provider, SP) 목록을 관리하는 권한을 가진 계정.

---

## 2. 세부 기능 요구사항 (Functional Requirements)

### 2.1 통합 인증 및 SSO 기능 (Authentication & Single Sign-On)
*   **FR-1. 중앙 집중식 로그인 (Login):** 사용자는 IdP(SSO 애플리케이션)에서 제공하는 통합 로그인 페이지에 사번(Employee ID)과 비밀번호를 입력하여 로그인한다.
*   **FR-2. SSO 토큰 발행 및 세션 공유 (Token Generation):** 성공적인 로그인 시, OAuth 2.0 또는 OIDC(OpenID Connect) 표준에 기반한 통합 인증 토큰(JWT)과 세션을 발행하여, 연동된 다른 사내 애플리케이션 접근 시 자동으로 인증을 처리한다.
*   **FR-3. 다중 요소 인증 (MFA/2FA):** 외부망(사외 IP 등) 접근 시 또는 고위험 애플리케이션 접근 시, 이메일/문자/OTP 앱을 통한 2차 인증을 필수로 요구한다.
*   **FR-4. 통합 로그아웃 (Single Log-Out, SLO):** 사용자가 어느 한 애플리케이션에서 로그아웃하면, 연동된 모든 애플리케이션의 인증 세션도 함께 만료되어야 한다.
*   **FR-5. 애플리케이션 접속 IP 제어 (IP Allowlisting):** 특정 연동 애플리케이션(SP)의 경우, 사전에 등록된 허용 IP 대역(Allowlist)에서 접근하는 사용자만 SSO 인가 및 접속이 가능하도록 제한하는 기능을 제공한다.

### 2.2 계정 및 데이터 동기화 정책 (Account & Data Sync Policy)
*   **FR-6. 계정 등록 및 데이터 동기화 (User Creation & Sync):**
    *   일반 사용자를 위한 회원가입 기능은 제공하지 않는다.
    *   최고 관리자 권한으로 특정 신규 계정(협력사 등)을 직접 등록하고 관리할 수 있는 API 기능을 제공한다.
    *   모든 사원 데이터의 기준(Source 단일 출처)은 자체 가입이 아닌 별도의 '인사 시스템의 데이터 파일(CSV 등)'로 지정한다. 
    *   이 파일을 통해 운영 환경에서는 매일 특정 시간(배치)에 DB를 동기화하고, 테스트 환경의 경우 서버 초기 실행 시 무조건 한 번 파일을 로드하여(On-load Sync) 읽어들이도록 분기 처리한다.
*   **FR-7. 계정 잠금 정책 (Account Lockout):** 연속 5회 로그인 실패 시 즉각적으로 해당 계정을 잠금 처리(Locked out)하고 이메일로 알림을 전송한다.
*   **FR-8. 비밀번호 수명 및 초기화:** 비밀번호는 매 90일마다 변경을 요구하며, 로그인 전 "비밀번호 찾기" 기능을 통해 본인 인증(이메일/SMS) 후 임시 비밀번호를 발급하거나 재설정 링크를 제공한다.
*   **FR-9. 동시 접속 제어:** 동일 계정으로 여러 기기(혹은 동일 브라우저의 다른 세션)에서 동시 접속하는 것을 정책에 따라 허용/차단할 수 있어야 하며, 기본값은 '이전 세션 로그아웃'으로 한다.

### 2.3 시스템 관리자 기능 (Admin & Dashboard)
*   **FR-10. 실시간 세션 모니터링:** 관리자는 현재 활성화된 SSO 세션(접속자, 접속 IP, 사용 중인 애플리케이션)을 대시보드에서 조회할 수 있다.
*   **FR-11. 세션 강제 종료:** 비정상적인 접근이 감지되거나 퇴사/보안 사고 발생 시, 관리자는 특정 사용자나 전체 사용자의 세션을 강제로 폐기(Revoke)할 수 있다.
*   **FR-12. 연동 애플리케이션 (SP) 및 IP 관리:** SSO 로그인을 사용할 사내 파트너 애플리케이션(Service Provider)의 인증 키(Client ID/Secret)를 발급, 갱신, 회수하며, 각 애플리케이션별로 접근을 허용할 IP 대역(Allowlist)을 설정하는 기능을 제공한다.

## 3. 비기능 요구사항 (Non-Functional Requirements)

*   **보안 및 인프라 설정 (Security & Config Isolation):**
    *   SAML 2.0 혹은 OIDC(OAuth 2.0) 등의 업계 표준 프로토콜을 준수해야 한다.
    *   인증 정보가 담긴 토큰(Access Token 등)은 반드시 HTTPS를 통해서만 전송되며, XSS/CSRF 공격의 근본적 차단을 위해 웹 클라이언트 스크립트에 노출되지 않는 **보안 쿠키(`HttpOnly`, `Secure`, `SameSite`) 형태**로 발급하여 전달/관리한다.
    *   인사 데이터 파일 위치, 관리자 계정 정보, 데이터베이스 연결 정보 등 민감한 자원들은 소스코드에 하드코딩하지 않으며, 보안 원칙에 따라 환경별(`테스트 .env.test`, `운영 .env.production`)로 반드시 분리하여 관리한다.
*   **고가용성 (High Availability) 및 성능:**
    *   모든 사내 시스템의 병목(Single Point of Failure)이 되지 않도록 다중 서버(Scale-out) 구조와 Redis 등을 활용한 글로벌 세션 저장소를 구성해야 한다. 
    *   SSO API의 평균 응답 시간은 200ms 이내여야 한다.
